Apifox的使用

2024-01-24

Apifox简介 1.为什么要使用Apifox 在当前行业发展背景下，绝大部分项目都是基于 前后端分离 的架构进行的，由前后端、测试、运维等不同的团队共同开发，那么团队之间能否很好的 协同合作 无疑直接决定着项目的最终效果。而协作其实也是大多数开发团队的痛点，为了便于团队协作，也出现了一些研发管理工具诸如 Jira、Teambition，应用也比较广泛，它们主要将功能聚焦于项目管理、代码托管、缺陷跟踪等环节。 但是在实际开发流程中，团队之间的协同是很低效的，比如需求评审结束之后，前后端进行接口定义并派一名工程师来定义 API 文档(一般使用 Swagger)；随后进入开发流程，后端按照 API 文档开发接口，在调试时还需按照文档在 Postman 中重新定义一遍；在后端开发出接口之前，前端如果需要开发就得按..
